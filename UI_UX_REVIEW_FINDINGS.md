# UI/UX-ревью: находки и статус

Аккуратное, точечное ревью интерфейса (дизайн-система **ORDO ARCANUM**, тёмный гримуар/кодекс).
Главное правило: **сохранять существующий стиль** — без новых цветов/шрифтов/библиотек, только
существующие токены `index.css` и примитивы `.ao-*`. Только точечные фиксы реальных проблем,
без редизайна. Приоритет: A) доступность → B) консистентность → C) состояния интерфейса →
D) адаптивность → E) иерархия → F) микровзаимодействия.

---

## A. Доступность — ✅ СДЕЛАНО (глобально, в `src/index.css`)

- **Focus-visible.** Был всего один тип `:focus` (только инпуты), на кнопках/вкладках/навигации
  designed-фокуса не было. Добавлен централизованный `:focus-visible` outline (`2px var(--gold-pale)`,
  offset 2px) для button/a/[role]/[tabindex] и кастомных контролов; текстовые инпуты исключены
  (у них уже box-shadow-кольцо — чтобы не было двойного индикатора).
- **Контраст.** `--ink-faint #6a614f` (≈2.9:1, проваливает AA) заменён на `--ink-quiet #968c75`
  (≈5.3:1) в `.ao-tab`, `.ao-table th`, `.ao-stat-label`, `.ao-empty-body`, `.wiz-rail-btn`.
- **prefers-reduced-motion** — глобальный media-блок гасит анимации/переходы.
- **Touch-таргеты** — `@media (pointer: coarse)` поднимает иконочные кнопки/вкладки до 44px (`.ao-btn--sm` 40px).

## B. Консистентность токенов — ✅ СДЕЛАНО (`src/index.css`)

- **Шкала шрифтов.** Добавлены `--t-nano: 10px`, `--t-pico: 9px`; литералы `font-size: 10px` (9 мест)
  и `9px` (4 места) переведены на токены.
- **Цвета.** Добавлены `--ember-pale: #d8896a`, `--moss: #7a9866`; повторяющийся хардкод-hex
  заменён на токены (в т.ч. `.ao-modtag.is-pos/.is-neg` через `color-mix`).

## D. Адаптивность — ✅ СДЕЛАНО (`src/index.css`)

- `100dvh`-фолбэки после `100vh` в `.ao-shell`, `.auth-split`; `.wiz-shell` → `calc(100dvh - 140px)`;
  диалог в media-query → `max-height: calc(100dvh - 32px)`. Чинит обрезание контента под мобильными
  адресными барами.

## C. Состояния интерфейса (loading / empty / error) — 🔶 ЧАСТИЧНО

### Эталон
`src/pages/gm/campaigns/CampaignListPage.tsx` — образцовый паттерн всех трёх состояний:
скелетоны (`.ao-panel .ao-frame .ao-breathe` + `.ao-ph`) → блок ошибки с кнопкой «Повторить»
(`refetch()`) → `EmptyVault` для пустого списка.

### Картина по проекту (на момент ревью)
- **loading** — обрабатывается широко: ~63 файла с `isLoading/isPending/isFetching`.
- **empty** — есть общий примитив `EmptyVault` (`src/components/ordo/EmptyVault.tsx`, класс `.ao-empty`),
  используется в ~20 файлах.
- **error** — **главный пробел:** на `isError`/`error` реагируют только ~11 файлов:
  `EditDoctrinePage`, `HomebrewLibraryPage`, `MonsterDetailPage`, `MonsterFormPage`,
  `BalanceManagementPage`, `BestiaryMonstersPage`, `CampaignBestiaryPage`, `CampaignListPage`,
  `MarketplacePage`, `CharacterResourcesPage`, `CharacterWalletPage`.
  → ~50 страниц грузят данные, но при сбое запроса не показывают ошибку (часто рендерят пустое
  состояние, вводя в заблуждение «данных нет» вместо «запрос упал»).

### Сделано в этом батче
**`CrudTable` (общая админ-таблица) — починен реальный баг.** Обрабатывала loading и empty, но не error:
при сбое `data === undefined` → ветка «пусто» → ложное «Записей не найдено. Нажмите „Добавить“…».
- `src/components/admin/CrudTable.tsx`: добавлены необязательные пропсы `isError?` / `onRetry?`
  (обратносовместимо) и ветка ошибки **в собственном Tailwind/shadcn-стиле таблицы**
  (`text-destructive`, кнопка `variant="outline"` «Повторить»). Родной вид таблицы не трогал.
- i18n: добавлен `cmp2.crud.loadError` (RU/EN) в `src/i18n/dict/components2.ts`; для кнопки
  переиспользован существующий `common.retry`.
- Подключены все 3 потребителя: `StatTypesPage`, `ItemTypesPage`, `CharacterClassesPage`
  (берут `isError`/`refetch` из react-query-хука).
- ⚠️ Ветка ошибки видна только при реальном падении запроса — нужна ручная/headless-невозможная
  визуальная проверка (например, временно «уронив» эндпоинт).

### Остаётся по C (пообъектно, требует визуальной проверки)
Error-состояния на ~50 бестейбловых страницах. Каждая со своей вёрсткой → не делается слепым
массовым рерайтом. Возможные пути (на выбор):
1. **Переиспользуемый компонент `ErrorAltar`** по образцу `EmptyVault` (`.ao-empty`-стилистика:
   сигил + заголовок + текст + кнопка «Повторить»), затем подключать пошагово по группам.
2. **Пообъектные проходы** по группам: начать с кампанийных list/detail (где `EmptyVault` уже есть,
   а блока ошибки нет), затем homebrew, затем бестиарий и т.д.

## Осознанно НЕ трогал и почему
- **Визуальный стиль `CrudTable`** — легаси на Tailwind/shadcn, а не `.ao-*`. Перевод на токены/`EmptyVault`
  = редизайн всего админ-раздела (против «сохранять стиль»). Ветка ошибки сделана в его же стиле.
- **Спекулятивный примитив `.ao-input.is-error`** — ошибки форм обрабатываются по-компонентно,
  общего класса нет; не плодим под гипотетику.
- **`min-height: 100vh` в модуле бестиария** — приватные стили страниц, низкий импакт; точечно при
  рефакторинге этих страниц.
- **Combat-экраны (`/combat-preview/*`)** — FE опережает BE, нет точки входа; отложены (см. `INLINE_CSS_REFACTOR_PLAN.md`).
