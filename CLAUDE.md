## Конвенция по стилям (CSS) — ОБЯЗАТЕЛЬНО

В проекте идёт целенаправленный уход от inline-стилей. **Не добавляй новые
`style={{…}}`** и приводи код к дизайн-системе.

### Где что лежит
- **Дизайн-токены** (цвета, шрифты, отступы, тени) — CSS-переменные в `src/index.css`
  (`--gold`, `--abyss`, `--font-display`, `--s-4` …). Всегда используй их, не хардкодь значения.
- **Общие примитивы** — классы `.ao-*` (и `.wiz-*`, `.forge-*`, `.cb-*`) в `src/index.css`.
  Сначала проверь, есть ли подходящий класс, прежде чем плодить новый.
- **Частные стили компонента/страницы** — CSS Modules: файл `Имя.module.css` рядом с компонентом,
  импорт `import s from './Имя.module.css'` и `className={s.foo}`.

### Правила
1. **Никаких новых `style={{…}}`** для статичных стилей. Выноси в класс (`.ao-*` или `*.module.css`).
2. **Inline допустим ТОЛЬКО для реально вычисляемых значений:** ширина бара `width: \`${pct}%\``,
   динамический `gridTemplateColumns`, координаты позиционирования. «Динамика из набора токенов» →
   CSS-переменная + класс: `style={{ '--fill': pct }}` и класс, читающий `var(--fill)`.
3. **Состояния — через классы-модификаторы**, а не тернарники в `style`:
   `className={cn('block', active && 'is-active')}` (см. `.ao-tab.is-active`, `.wiz-rail-btn.is-active`).
4. **Ховер/фокус — только в CSS** (`:hover`, `:focus`). **Запрещено** рисовать ховер через
   `onMouseEnter`/`onMouseLeave` с мутацией `e.currentTarget.style`.
5. **Не вводи Tailwind-утилиты массово.** Используется собственная `.ao-*` система — доращивай её.
   (Tailwind присутствует в зависимостях, но в разметке его широко не применяем.)
6. Значения цвета/отступа бери из токенов `index.css`, не из «магических» чисел и hex.

### Утилиты раскладки (вместо inline-flex)
Для частых статичных flex-раскладок используй готовые утилиты из `index.css`
вместо `style={{ display:'flex', … }}`. Композируются через пробел:
`className="ao-row ao-between ao-gap-12"`.
- `.ao-row` (flex + центр по вертикали), `.ao-row-start/-end/-baseline`, `.ao-col` (колонка).
- `.ao-center` (центр по обеим осям), `.ao-between`, `.ao-justify-end`, `.ao-justify-center`.
- `.ao-wrap` (`flex-wrap`), `.ao-grow` (`flex:1`).
- `.ao-gap-N` — `gap: N px` (N ∈ 2,4,6,8,10,12,14,16,18,24).
Реальную динамику (вычисляемая ширина, динамический `gridTemplateColumns`) оставляй inline.

### Активный рефакторинг
План поэтапного выноса inline-CSS и текущий прогресс — в **`INLINE_CSS_REFACTOR_PLAN.md`**
(корень репозитория). Работаем по одной задаче, после визуального теста пользователем
пункт отмечается выполненным. Если правишь стили — сверяйся с планом и не ломай уже вынесенное.

## Команды

- Dev-сервер: `npm run dev` (Vite)
- Сборка: `npm run build`
- Линт: `npm run lint`
- Тайпчек: `npx tsc -b --force` — **важно:** обычный `tsc --noEmit` тут молча ничего не проверяет.

## Стек

React 18 + Vite + TypeScript, react-router v6 (`src/router.tsx`), zustand (стор),
@tanstack/react-query, Radix UI, i18n через `src/i18n/I18nContext`, WebSocket (STOMP/SockJS).

## Архитектура UI (кратко)

- `src/components/layout/AppLayout.tsx` — постоянная оболочка: боковая панель (`rail`) /
  мобильный `drawer` / `header` + `<Outlet/>`. Страницы рендерятся внутри Outlet.
- Полноэкранные боевые экраны (`/combat-preview/*`) вынесены **вне** `AppLayout` намеренно.
- Не предлагай iframe / «окно в окне» для разделения UI — нужный эффект уже даёт
  persistent-shell + nested routes (Outlet). Для усиления — вложенные layout’ы и route-modals.
